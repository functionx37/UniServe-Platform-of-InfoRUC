## 1. 项目概览与核心约束

* 技术栈: 微信小程序 (学生端) + Vue/React (管理端) + Spring Boot/FastAPI (后端) + Kingbase (数据库)。 

## 2. 核心分工与详细执行任务

### 成员 A: 后端架构、权限与安全

职责：搭建系统骨架，守住安全大门。 

* 基础架构：搭建 Spring Boot 环境，配置 Kingbase 驱动。写一个统一的 JSON 返回类。 
* 权限校验：实现四级权限逻辑（领导、老师、骨干、学生）。编写拦截器（Interceptor）校验 role_id。 
*	审批流逻辑（F-13）：开发请假、证明等申请的后台 API，控制状态流转（待审 -> 通过/驳回）。 
*	数据脱敏：在返回给前端的敏感数据（如手机号）上，用代码实现星号遮蔽（如 138****0000）。

### 成员 B: AI 算法、学业分析与文档处理 (逻辑专家)

职责：攻克“最费脑子”的算法逻辑与文件处理。 

*	智能问答（F-01/02）：基于 RAG 模式实现。先将学院政策 PDF 切片存入向量库，AI 仅根据匹配到的片段回答，杜绝瞎编。 
*	PDF/Excel 解析（F15）：编写解析脚本，提取学生上传成绩单中的课程名和学分。 
*	学业分析算法（F16/18）：编写逻辑比对“已修课程”与“培养方案”，计算学分缺口并给出选课建议。 
*	证明生成（F12）：将学生信息自动套用到 PDF 模板中，生成可下载的电子证明文件。 

### 成员 C: 微信小程序端 (学生端前端)

职责：负责学生直接接触的手机界面。 

*	页面开发：画出智能问答、学业进度(F-05)、申请列表等 5 个核心页面。 
*	可视化效果：使用 CSS 动画实现党团进度“呼吸灯”效果。 
*	接口联调：使用 wx.request 请求 A 的 IP 地址。开发初期先用“假数据”占位，等 A 写好接口后再替换。 
*	文件上传：实现成绩单 PDF 上传功能并传给后端。 

### 成员 D: PC 管理后台与全栈辅助 (全栈/QA)

职责：负责老师用的管理界面，并把控整体交付质量。 

*	后台页面：使用 Vue/React 搭建侧边栏结构的管理员界面。 
*	批量操作(F-14)：实现政策、通知的 Excel 批量导入导出功能。 
*	标签推送(F-10/11)：根据年级、专业标签，实现精准的信息分发 API 调用。 
*	质量保证：编写 Dockerfile 确保一键部署；使用 Postman 进行接口测试，编写压力测试报告。 

## 3. 核心接口与数据库约定

### 最小接口通信约定

所有接口必须返回统一的 JSON 格式：

```json
{
  "success": true, 
  "message": "操作成功",
  "data": { } 
}
```

### 关键 API 路径预览

| 功能     | 方法 | 路径             | 关键字段                |
| -------- | ---- | ---------------- | ----------------------- |
| 登录     | POST | /auth/login      | username, password      |
| AI 提问  | POST | /ai/ask          | question -> answer      |
| 学业查询 | GET  | /academic/status | total_credits, gap      |
| 审批操作 | POST | /admin/audit     | id, action(pass/reject) |

### 数据库核心表 (Kingbase 适配)
*	users: id, username, role_id(1-4), encrypted_id_card, nickname
*	applications: id, user_id, type, status(0-2), created_at

## 4. 协作“三步走”战略

* 第一阶段 (接口先行): 成员 A 连通数据库，并在群里发布具体的 API 路径 和 字段名。

* 第二阶段 (并行开发):
  * B 钻研 AI 调优。
  * C 和 D 在本地用“假数据”画页面，并勾选“不校验合法域名”。
  * A 编写包含权限控制的业务逻辑。

* 第三阶段 (合体联调): C 和 D 将代码中的 http://localhost 改为 http://[A的IP]，进行全流程测试。

## 5. 最终验收避坑清单
*	权限拦截：学生账号绝对进不去 /admin 页面。 
*	隐私处理：所有显示身份证或手机号的地方必须带 * 号。 
*	AI 回答：问政策相关问题时，AI 必须能引用文档原文。 
*	国产适配：所有的 SQL 操作在 Kingbase 环境下无报错。 
*	部署交付：Docker 镜像能一键启动，README 写清了测试账号。
