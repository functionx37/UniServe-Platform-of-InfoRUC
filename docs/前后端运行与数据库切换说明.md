# 前后端运行与数据库切换说明

本文档说明当前仓库中管理员前端、后端的启动方式，以及如何先用开源数据库替代金仓，把项目尽快跑起来。

## 1. 当前项目状态

- 管理员前端目录：`admin-frontend/`
- 后端目录：`backend/`
- 小程序目录：`miniprogram/`，本文档不展开
- 当前管理员前端主要使用本地 mock 数据，启动前端时不依赖后端接口
- 当前后端只有一个测试接口：`GET /api/test/hello`
- 当前后端代码里还没有真正访问数据库的业务 Mapper、Service、Entity，因此数据库替换成本很低

## 2. 环境要求

### 前端

- Node.js 18 及以上，建议使用 Node.js 20 或 22
- npm，随 Node.js 一起安装

### 后端

- JDK 17
- Maven Wrapper 已包含在仓库中，优先使用 `mvnw.cmd`

## 3. 管理员前端启动方法

### 3.1 安装依赖

在项目根目录打开终端，执行：

```powershell
cd admin-frontend
npm install
```

### 3.2 启动开发服务器

```powershell
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

### 3.3 前端常见问题

#### 问题 1：`vite is not recognized`

原因：还没有安装依赖。

解决：

```powershell
npm install
```

#### 问题 2：前端能打开，但没有联调后端

这是当前仓库的正常现象。`admin-frontend/src/api/adminApi.ts` 里现在使用的是本地 mock 数据和 `localStorage`，不是实际后端接口。

也就是说：

- 前端可以单独运行
- 就算后端没启动，前端页面也能展示和操作演示数据

## 4. 后端启动方法

### 4.1 先确认 Java 环境

在终端执行：

```powershell
java -version
```

如果提示找不到 `java`，说明本机还没有安装 JDK，或者没有把 JDK 的 `bin` 目录加入环境变量 `PATH`。

建议安装：

- JDK 17

### 4.2 当前后端的启动命令

进入后端目录后执行：

```powershell
cd backend
.\mvnw.cmd spring-boot:run
```

或者先打包再运行：

```powershell
cd backend
.\mvnw.cmd clean package
java -jar .\target\backend-0.0.1-SNAPSHOT.jar
```

### 4.3 默认测试接口

启动成功后可访问：

```text
http://localhost:8080/api/test/hello
```

预期返回类似：

```json
{
  "success": true,
  "message": "操作成功",
  "data": "你好！人大信息学院后台服务已就绪。"
}
```

## 5. 为什么现在后端不一定能直接启动

当前 `backend/src/main/resources/application.yml` 中写死了金仓数据库配置：

```yml
spring:
  datasource:
    driver-class-name: com.kingbase8.Driver
    url: jdbc:kingbase8://localhost:54321/student_db
    username: system
    password: 123456
```

如果本机没有：

- 安装 JDK 17
- 安装并启动金仓数据库
- 创建 `student_db`
- 配好账号密码

那么后端启动时就可能失败。

## 6. 可以先换成开源数据库吗

可以，完全可以，而且当前阶段很适合这么做。

原因有两个：

- 当前后端还没有真正使用数据库表和复杂 SQL
- 当前只是在做基础启动和接口联通，先用开源数据库把开发链路跑通更合适

## 7. 推荐方案：先换 PostgreSQL

相比 MySQL，更推荐先换 PostgreSQL，原因是：

- 金仓和 PostgreSQL 生态更接近
- 后续如果再切回金仓，迁移成本通常比从 MySQL 切换更低
- JDBC URL、驱动使用方式也更接近

## 8. PostgreSQL 替换步骤

### 8.1 安装 PostgreSQL

本机安装一个 PostgreSQL 14 或 15 都可以。

建议先创建：

- 数据库名：`student_db`
- 用户名：`postgres`
- 密码：`123456`
- 端口：`5432`

### 8.2 修改 Maven 依赖

编辑 `backend/pom.xml`：

1. 删除金仓驱动依赖
2. 增加 PostgreSQL 驱动依赖

把这段：

```xml
<dependency>
    <groupId>com.kingbase8</groupId>
    <artifactId>kingbase8</artifactId>
    <version>8.6.0</version>
</dependency>
```

替换为：

```xml
<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <scope>runtime</scope>
</dependency>
```

### 8.3 修改数据源配置

编辑 `backend/src/main/resources/application.yml`，改成：

```yml
spring:
  datasource:
    driver-class-name: org.postgresql.Driver
    url: jdbc:postgresql://localhost:5432/student_db
    username: postgres
    password: 123456

mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
    map-underline-to-camel-case: true
```

### 8.4 重新启动后端

```powershell
cd backend
.\mvnw.cmd spring-boot:run
```

## 9. 更快的临时方案：先不连数据库也能跑

如果当前目标只是让后端服务先起来，而不是立刻访问数据库，那么还有一个更快的临时方案：

- 先注释掉 `application.yml` 里的 `spring.datasource` 配置
- 保留 `spring-boot-starter-web`
- 直接启动后端

由于当前代码里只有测试接口，没有真正的数据库读写逻辑，这种做法通常也能让服务先跑起来。

适用场景：

- 先验证 Java、Spring Boot、接口访问链路
- 先给前后端联调一个基础服务
- 数据库设计和建表还没定稿

不适用场景：

- 已经开始写 Mapper、增删改查接口
- 需要真实存储数据

## 10. 建议的最短落地顺序

建议按下面顺序推进：

1. 先安装 JDK 17，确保后端基础环境可用
2. 前端先独立跑起来，确认页面无问题
3. 后端先用“不连数据库”或 PostgreSQL 方式跑通
4. 用 `GET /api/test/hello` 验证后端服务可访问
5. 后续真正开始写数据表和业务接口时，再决定是否切回金仓

## 11. 一句话结论

- 前端现在可以独立跑起来
- 后端当前主要受 JDK 和数据库配置影响
- 现阶段完全可以先不用金仓，推荐先换 PostgreSQL，或者先临时去掉数据源配置，把服务跑通
