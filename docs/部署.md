# 部署说明

本文档说明当前仓库的数据库、后端、前端启动方式，以及前端修改后的更新发布命令。

当前仓库目录：

- 管理后台前端：`admin-frontend/`
- 后端服务：`backend/`
- 小程序：`miniprogram/`

## 1. 当前默认方案

当前项目默认使用：

- 前端：React + Vite
- 后端：Spring Boot 3.x + MyBatis-Plus
- 数据库：PostgreSQL

默认端口与路径：

- PostgreSQL：`5432`
- 后端：`8081`
- 前端发布目录：`/var/www/uniserve-admin`

说明：

- 当前管理后台前端已经接到真实后端接口
- 前端发布时使用仓库自带脚本 `admin-frontend/scripts/deploy-to-nginx.sh`
- Nginx 会直接提供前端静态文件，并将 `/auth/*`、`/admin/*`、`/files/*` 反向代理到 `127.0.0.1:8081`

## 2. 环境要求

### 2.1 前端环境

- Node.js 20 及以上
- npm
- nginx
- sudo

### 2.2 后端环境

- JDK 17
- Maven Wrapper，仓库已自带

### 2.3 数据库环境

- PostgreSQL 14、15、16 均可

默认数据库参数如下：

- 数据库名：`student_db`
- 用户名：`postgres`
- 密码：`123456`
- 端口：`5432`

## 3. 数据库启动与初始化

后端启动前，必须先准备 PostgreSQL 并导入初始化脚本。

### 3.1 Linux 下安装 PostgreSQL

Ubuntu / Debian 可参考：

```bash
sudo apt update
sudo apt install -y postgresql postgresql-client
```

### 3.2 启动 PostgreSQL

```bash
sudo systemctl enable --now postgresql
```

如果只是临时启动，也可以使用：

```bash
sudo systemctl start postgresql
```

### 3.3 创建数据库并设置密码

```bash
sudo -u postgres psql -c "ALTER USER postgres WITH PASSWORD '123456';"
sudo -u postgres createdb student_db
```

如果提示数据库已存在，可以忽略 `createdb` 的报错。

### 3.4 导入初始化脚本

```bash
PGPASSWORD=123456 psql -h localhost -U postgres -d student_db -f /home/user/UniServe-Platform-of-InfoRUC/backend/src/main/resources/sql/init.sql
```

### 3.5 验证数据库是否正常

查看数据表：

```bash
PGPASSWORD=123456 psql -h localhost -U postgres -d student_db -c "\dt"
```

查看用户初始数据：

```bash
PGPASSWORD=123456 psql -h localhost -U postgres -d student_db -c "select username, role_id, student_no, real_name from users order by id;"
```

正常情况下至少可以看到：

- 管理员：`teacher01`
- 学生：`20260001`

## 4. 后端启动

### 4.1 默认数据库配置

当前后端默认读取的是 `backend/src/main/resources/application.yml` 中的数据源配置：

```yml
spring:
  datasource:
    driver-class-name: ${DB_DRIVER_CLASS_NAME:org.postgresql.Driver}
    url: ${DB_URL:jdbc:postgresql://localhost:5432/student_db}
    username: ${DB_USERNAME:postgres}
    password: ${DB_PASSWORD:123456}
```

也就是说，如果你不额外传环境变量，后端会默认连接本机 PostgreSQL。

### 4.2 启动命令

进入后端目录后执行：

```bash
cd /home/user/UniServe-Platform-of-InfoRUC/backend
bash ./mvnw spring-boot:run
```

如果是在受限环境中遇到 Maven 本地缓存目录权限问题，可改用：

```bash
cd /home/user/UniServe-Platform-of-InfoRUC/backend
MAVEN_USER_HOME=/tmp/maven-user-home bash ./mvnw spring-boot:run -Dmaven.repo.local=/tmp/maven-repo
```

如果提示 `Permission denied`，先执行：

```bash
chmod +x /home/user/UniServe-Platform-of-InfoRUC/backend/mvnw
```

### 4.3 启动成功后的验证命令

后端默认端口为 `8081`，可用下面命令验证：

```bash
curl http://localhost:8081/api/test/hello
```

预期返回示例：

```json
{
  "success": true,
  "message": "操作成功",
  "data": "你好！人大信息学院后台服务已就绪。"
}
```

### 4.4 测试账号

数据库初始化完成后，可使用以下账号：

- 管理员：`teacher01 / 123456`
- 学生：`20260001 / 张三`

说明：

- 管理员端是账号密码登录
- 学生端目前是学号加姓名登录

## 5. 前端发布到公共目录

当前仓库已经提供前端发布脚本：

```text
/home/user/UniServe-Platform-of-InfoRUC/admin-frontend/scripts/deploy-to-nginx.sh
```

该脚本会自动执行以下操作：

1. 安装前端依赖
2. 构建前端
3. 清空公共目录 `/var/www/uniserve-admin`
4. 将 `dist` 发布到公共目录
5. 安装 Nginx 配置
6. 校验并重载 Nginx

### 5.1 首次发布命令

```bash
cd /home/user/UniServe-Platform-of-InfoRUC/admin-frontend
chmod +x ./scripts/deploy-to-nginx.sh
./scripts/deploy-to-nginx.sh
```

### 5.2 前端修改后重新更新命令

前端代码改完后，重新执行同一个脚本即可：

```bash
cd /home/user/UniServe-Platform-of-InfoRUC/admin-frontend
./scripts/deploy-to-nginx.sh
```

### 5.3 脚本默认使用的路径

脚本默认使用：

- 前端公共目录：`/var/www/uniserve-admin`
- Nginx 配置目标：`/etc/nginx/conf.d/uniserve-admin.conf`

如需覆盖默认路径，可在执行时传环境变量：

```bash
cd /home/user/UniServe-Platform-of-InfoRUC/admin-frontend
PUBLIC_DIR=/var/www/my-admin \
NGINX_CONF_TARGET=/etc/nginx/conf.d/my-admin.conf \
./scripts/deploy-to-nginx.sh
```

### 5.4 发布完成后的访问地址

默认 Nginx 配置中写的是：

- 静态页面根路径：`http://10.10.0.9/`
- 后端代理：`http://127.0.0.1:8081`

所以发布成功后可直接访问：

```text
http://10.10.0.9/
```

### 5.5 发布前请确认

- 服务器已安装 `node`、`npm`、`nginx`
- 当前用户具有 `sudo` 权限
- 后端已经运行在 `8081`
- 如果后端端口不是 `8081`，请同步修改 `admin-frontend/deploy/nginx/uniserve-admin.conf`

### 5.6 推荐更新顺序

服务器上最推荐的更新顺序如下：

1. 启动 PostgreSQL
2. 确认 `student_db` 已创建并已导入初始化 SQL
3. 启动后端
4. 用 `curl http://localhost:8081/api/test/hello` 验证后端
5. 前端代码修改完成后，执行 `./scripts/deploy-to-nginx.sh`
6. 通过 Nginx 访问前端页面并验证接口联通

## 7. 切回金仓的方法

如果后续需要切回金仓，不必改 Java 代码，只需要在启动前覆盖环境变量：

```bash
export DB_DRIVER_CLASS_NAME=com.kingbase8.Driver
export DB_URL=jdbc:kingbase8://localhost:54321/student_db
export DB_USERNAME=system
export DB_PASSWORD=123456
```

注意：

- 如果要真正切回金仓，还需要保证本机已安装金仓数据库
- 还需要准备对应的 JDBC 驱动依赖和可用数据库实例

## 8. 常见问题

### 8.1 `java: command not found`

原因：

- 没有安装 JDK 17
- 或者 `PATH` 没配置好

可先检查：

```bash
java -version
```

### 8.2 `psql: command not found`

原因：

- PostgreSQL 客户端未安装

Ubuntu / Debian 可执行：

```bash
sudo apt install -y postgresql-client
```

### 8.3 后端启动失败，提示无法连接数据库

重点检查：

- PostgreSQL 是否已启动
- `student_db` 是否已创建
- 用户名密码是否与 `application.yml` 一致
- 初始化 SQL 是否已导入

可先用下面命令测试数据库连接：

```bash
PGPASSWORD=123456 psql -h localhost -U postgres -d student_db -c "\dt"
```

### 8.4 后端启动时出现 Maven 权限问题

如果是在 Linux 环境中直接运行 `./mvnw` 失败，可先授权：

```bash
chmod +x /home/user/UniServe-Platform-of-InfoRUC/backend/mvnw
```

如果仍有权限或缓存目录问题，可改用：

```bash
cd /home/user/UniServe-Platform-of-InfoRUC/backend
MAVEN_USER_HOME=/tmp/maven-user-home bash ./mvnw spring-boot:run -Dmaven.repo.local=/tmp/maven-repo
```

### 8.5 前端发布脚本执行失败

重点检查：

- 是否已安装 `node`
- 是否已安装 `npm`
- 是否已安装 `nginx`
- 当前用户是否具备 `sudo` 权限
- `/etc/nginx/conf.d/` 和 `/var/www/` 是否允许写入

也可以先手动检查：

```bash
node -v
npm -v
nginx -v
sudo nginx -t
```
