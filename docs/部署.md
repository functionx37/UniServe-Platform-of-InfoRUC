# 部署说明

本文档说明当前仓库在本地开发环境中的部署与启动方式，覆盖以下内容：

- 前端启动
- 后端启动
- PostgreSQL 数据库准备与初始化
- 常见验证方式与问题说明

当前仓库目录：

- 管理后台前端：`admin-frontend/`
- 后端服务：`backend/`
- 小程序：`miniprogram/`

## 1. 当前默认方案

当前项目本地开发默认使用：

- 前端：Vite
- 后端：Spring Boot 3.x + MyBatis-Plus
- 数据库：PostgreSQL

说明：

- 当前后端默认本地开发数据库已切换为 PostgreSQL
- 后续如果需要切回金仓，可以通过环境变量覆盖数据库配置
- 管理后台前端目前仍包含部分本地 mock 数据，因此前端可以单独启动
- 后端已经可以独立连接 PostgreSQL 并启动

## 2. 环境要求

### 2.1 前端环境

- Node.js 18 及以上，建议 Node.js 20 或 22
- npm

### 2.2 后端环境

- JDK 17
- Maven Wrapper，仓库已自带

### 2.3 数据库环境

- PostgreSQL 14、15、16 均可

默认数据库参数如下：

- 数据库名：`student_db`
- 用户名：`postgres`
- 密码：`123456`
- 端口：`5432`

## 3. 数据库准备

后端启动前，必须先准备数据库并导入初始化脚本。

### 3.1 Linux 下安装 PostgreSQL

Ubuntu / Debian 可参考：

```bash
sudo apt update
sudo apt install -y postgresql postgresql-client
```

### 3.2 启动 PostgreSQL

```bash
sudo systemctl enable --now postgresql
```

### 3.3 创建数据库并设置密码

```bash
sudo -u postgres psql -c "ALTER USER postgres WITH PASSWORD '123456';"
sudo -u postgres createdb student_db
```

如果提示数据库已存在，可以忽略 `createdb` 的报错。

### 3.4 导入初始化脚本

```bash
PGPASSWORD=123456 psql -h localhost -U postgres -d student_db -f /home/user/UniServe-Platform-of-InfoRUC/backend/src/main/resources/sql/init.sql
```

### 3.5 验证数据库是否初始化成功

查看数据表：

```bash
PGPASSWORD=123456 psql -h localhost -U postgres -d student_db -c "\dt"
```

查看用户初始数据：

```bash
PGPASSWORD=123456 psql -h localhost -U postgres -d student_db -c "select username, role_id, student_no, real_name from users order by id;"
```

正常情况下至少可以看到：

- 管理员：`teacher01`
- 学生：`20260001`

## 4. 后端启动

### 4.1 默认数据库配置

当前后端默认读取的是 `backend/src/main/resources/application.yml` 中的数据源配置：

```yml
spring:
  datasource:
    driver-class-name: ${DB_DRIVER_CLASS_NAME:org.postgresql.Driver}
    url: ${DB_URL:jdbc:postgresql://localhost:5432/student_db}
    username: ${DB_USERNAME:postgres}
    password: ${DB_PASSWORD:123456}
```

也就是说，如果你不额外传环境变量，后端会默认连接本机 PostgreSQL。

### 4.2 启动命令

进入后端目录后执行：

```bash
cd /home/user/UniServe-Platform-of-InfoRUC/backend
bash ./mvnw spring-boot:run
```

如果提示 `Permission denied`，先执行：

```bash
chmod +x /home/user/UniServe-Platform-of-InfoRUC/backend/mvnw
./mvnw spring-boot:run
```

### 4.3 启动成功后的访问地址

后端默认端口为 `8081`，测试接口为：

```text
http://localhost:8081/api/test/hello
```

预期返回示例：

```json
{
  "success": true,
  "message": "操作成功",
  "data": "你好！人大信息学院后台服务已就绪。"
}
```

### 4.4 测试账号

数据库初始化完成后，可使用以下账号：

- 管理员：`teacher01 / 123456`
- 学生：`20260001 / 张三`

说明：

- 管理员端是账号密码登录
- 学生端目前是学号加姓名登录

## 5. 前端启动

### 5.1 安装依赖

```bash
cd /home/user/UniServe-Platform-of-InfoRUC/admin-frontend
npm install
```

### 5.2 启动开发服务器

```bash
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

### 5.3 当前前端与后端关系

当前管理后台前端仍有一部分页面使用本地 mock 数据和 `localStorage`，因此：

- 前端可以单独启动
- 后端不启动时，部分页面仍可展示
- 如果后续改成真实接口联调，则需要保证后端同步运行

## 6. 推荐启动顺序

建议按以下顺序操作：

1. 安装并启动 PostgreSQL
2. 创建 `student_db`
3. 导入 `backend/src/main/resources/sql/init.sql`
4. 启动后端
5. 访问 `http://localhost:8081/api/test/hello` 验证后端
6. 启动前端
7. 打开 `http://localhost:5173`

## 7. 切回金仓的方法

如果后续需要切回金仓，不必改 Java 代码，只需要在启动前覆盖环境变量：

```bash
export DB_DRIVER_CLASS_NAME=com.kingbase8.Driver
export DB_URL=jdbc:kingbase8://localhost:54321/student_db
export DB_USERNAME=system
export DB_PASSWORD=123456
```

注意：

- 如果要真正切回金仓，还需要保证本机已安装金仓数据库
- 还需要准备对应的 JDBC 驱动依赖和可用数据库实例

## 8. 常见问题

### 8.1 `java: command not found`

原因：

- 没有安装 JDK 17
- 或者 `PATH` 没配置好

可先检查：

```bash
java -version
```

### 8.2 `psql: command not found`

原因：

- PostgreSQL 客户端未安装

Ubuntu / Debian 可执行：

```bash
sudo apt install -y postgresql-client
```

### 8.3 前端报 `vite: command not found`

通常是因为依赖未安装：

```bash
cd /home/user/UniServe-Platform-of-InfoRUC/admin-frontend
npm install
```

### 8.4 后端启动失败，提示无法连接数据库

重点检查：

- PostgreSQL 是否已启动
- `student_db` 是否已创建
- 用户名密码是否与 `application.yml` 一致
- 初始化 SQL 是否已导入

可先用下面命令测试数据库连接：

```bash
PGPASSWORD=123456 psql -h localhost -U postgres -d student_db -c "\dt"
```

### 8.5 后端启动时出现 Maven 权限问题

如果是在 Linux 环境中直接运行 `./mvnw` 失败，可先授权：

```bash
chmod +x /home/user/UniServe-Platform-of-InfoRUC/backend/mvnw
```

如果仍有权限或缓存目录问题，可改用：

```bash
cd /home/user/UniServe-Platform-of-InfoRUC/backend
bash ./mvnw spring-boot:run
```

## 9. 一句话总结

本项目当前本地开发最推荐的方式是：

- 数据库使用 PostgreSQL
- 后端运行在 `8081`
- 前端运行在 `5173`
- 先启动数据库，再启动后端，最后启动前端
