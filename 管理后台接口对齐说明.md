# 管理后台接口对齐说明

本文档用于登记同学 D 管理后台已经使用到的接口，方便后续同学 A 或其他同学按同一套字段实现后端。

## 1. 统一返回格式

所有接口统一返回以下 JSON 结构：

```json
{
  "success": true,
  "message": "操作成功",
  "data": {}
}
```

说明：

- `success`：布尔值，表示本次请求是否成功。
- `message`：字符串，给前端展示的结果说明。
- `data`：具体业务数据。

## 2. 登录接口

### `POST /auth/login`

用途：

- 管理老师登录 PC 管理后台。

请求体：

```json
{
  "username": "teacher01",
  "password": "123456"
}
```

返回示例：

```json
{
  "success": true,
  "message": "登录成功",
  "data": {
    "token": "jwt-token",
    "role": "管理老师",
    "displayName": "李老师"
  }
}
```

字段说明：

- `token`：后续访问管理接口时放到 `Authorization: Bearer <token>`。
- `role`：用于前端展示和后端鉴权。
- `displayName`：顶部显示的老师姓名。

## 3. 仪表盘接口

### `POST /admin/dashboard`

用途：

- 获取后台首页统计卡片数据。

请求体：

```json
{
  "grade": "2022",
  "major": "计算机科学与技术",
  "identity": "全部"
}
```

返回示例：

```json
{
  "success": true,
  "message": "获取仪表盘数据成功",
  "data": {
    "pendingNotificationCount": 3,
    "targetStudentCount": 86,
    "recentDeliveryCount": 12,
    "latestImportSuccessRate": 96
  }
}
```

字段说明：

- `pendingNotificationCount`：待发布通知数量。
- `targetStudentCount`：按当前标签命中的学生人数。
- `recentDeliveryCount`：最近发送记录数。
- `latestImportSuccessRate`：最近一次导入成功率，单位为百分数整数。

## 4. 通知列表接口

### `GET /admin/notifications`

用途：

- 获取通知数据表内容。

返回示例：

```json
{
  "success": true,
  "message": "获取通知列表成功",
  "data": [
    {
      "id": "policy-001",
      "title": "2026年研究生国奖申报通知",
      "category": "奖助",
      "grade": "全部",
      "major": "全部",
      "channel": "站内消息、邮件",
      "publishAt": "2026-05-02 09:00",
      "status": "待发布"
    }
  ]
}
```

字段说明：

- `id`：通知主键。
- `title`：通知标题。
- `category`：通知分类，如奖助、党建、竞赛、就业、实习、通知。
- `grade`：目标年级。
- `major`：目标专业。
- `channel`：推送渠道，前端暂时按字符串展示。
- `publishAt`：发布时间。
- `status`：状态，如待发布、已发布。

## 5. 批量导入接口

### `POST /admin/import/notifications`

用途：

- 处理 Excel/CSV 的通知批量导入。

推荐请求方式：

- `multipart/form-data`
- 文件字段名：`file`

后端解析后返回：

```json
{
  "success": true,
  "message": "已导入24行，失败1行",
  "data": {
    "importSession": {
      "id": "import-001",
      "fileName": "活动通知模板.xlsx",
      "totalRows": 25,
      "successRows": 24,
      "failedRows": 1,
      "importedAt": "2026-05-02 20:10:00"
    },
    "notifications": [
      {
        "id": "policy-001",
        "title": "信息学院春季专场招聘会",
        "category": "就业",
        "grade": "2022",
        "major": "计算机科学与技术",
        "channel": "站内消息、邮件",
        "publishAt": "2026-05-05 14:00",
        "status": "待发布"
      }
    ]
  }
}
```

导入模板字段要求：

- `标题`
- `分类`
- `年级`
- `专业`
- `渠道`
- `发布时间`
- `状态`

校验规则：

- `标题` 必填
- `分类` 必填
- 其余字段为空时可按默认值补齐

## 7. 导出接口

### `GET /admin/export/notifications`

用途：

- 导出当前通知列表，建议后端直接返回 Excel 文件流。

建议参数：

- `grade`
- `major`
- `category`
- `status`

说明：

- 当前前端演示版是本地直接导出 Excel。
- 后续真实联调时，建议后端返回二进制文件流，前端只负责下载。

## 8. 推送预览接口

### `POST /admin/push/preview`

用途：

- 根据标签预览将被触达的学生。

请求体：

```json
{
  "grade": "2022",
  "major": "计算机科学与技术",
  "identity": "全部"
}
```

返回示例：

```json
{

  "message": "获取推送预览成功",
  "data": {
    "recipients": [
      {
        "id": "20221001",
        "name": "张雨晨",
        "grade": "2022",
```

## 9. 精准推送发送接口

### `POST /admin/push/send`

用途：

- 根据标签向命中对象发送站内消息、邮件等通知。

请求体：

```json
{
  "title": "2026届实习信息汇总推送",
  "content": "请相关同学于本周五前完成岗位志愿填报。",
  "grade": "2022",
  "major": "计算机科学与技术",
  "identity": "全部",
  "channels": ["站内消息", "邮件"]
}
```

返回示例：

```json
{
  "success": true,
  "message": "通知发送成功",
  "data": {
    "id": "delivery-001",
    "title": "2026届实习信息汇总推送",
    "audience": "2022 / 计算机科学与技术 / 全部",
    "channels": "站内消息、邮件",
    "sentAt": "2026-05-02 20:20:00",
    "count": 86,
    "status": "已发送"
  }
}
```

后端额外建议：

- 写入发送日志表。
- 保存操作人、发送时间、发送渠道、命中人数。
- 对 `0` 命中对象也返回成功，但 `status` 可记为 `无匹配对象`。

## 11. 鉴权建议

所有 `/admin/*` 路径建议校验以下内容：

- 用户已登录
- `role` 为 `学院领导` 或 `管理老师`
- 非管理员角色禁止访问导入、导出、推送发送等接口

请求头示例：

```text
Authorization: Bearer <token>
```

## 12. 当前前端实现说明

当前前端不是直接写死数据，而是已经通过统一 API 层工作：

- `admin-frontend/src/api/adminApi.ts`

这意味着后续只需要把该文件里的 Mock 实现替换成真实 `fetch/axios` 请求，页面层基本不用重写。
